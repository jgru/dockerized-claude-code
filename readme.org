#+title: Dockerized Claude Code

Run Claude Code in a container (also driven from Emacs) to provide
file system isolation and protect your secrets. =git= uses tokens from
each project's =.env=. Claude uses OAuth tokens retrieved via macOS'
=keychain= (fallback: read from an unencrypted file with the implied
security risks).

* Files

#+begin_example
bin/
  claude-docker.sh        Wrapper script -> ~/.local/bin/claude-docker
container/
  Dockerfile              Claude Code based on node 22
  entrypoint.sh           git tokens -> exec claude
  git-credential-token    Credential helper (reads tokens from env)
emacs/
  claude-code-ide-ext.el  Emacs configuration
.env.example              Template .env for projects
#+end_example

* Setup

#+begin_src sh
make install && make build
#+end_src

=install= copies container files to =~/.config/claude-devcontainer/= and the
wrapper to =~/.local/bin/claude-docker=. =build= builds the Docker image.

/Alternatively, do the following manually:/
#+begin_src bash
# 1. Copy container files
mkdir -p ~/.config/claude-devcontainer
cp container/Dockerfile container/entrypoint.sh container/git-credential-token \
   ~/.config/claude-devcontainer/
chmod +x ~/.config/claude-devcontainer/entrypoint.sh \
         ~/.config/claude-devcontainer/git-credential-token

# 2. Build image
docker build -t claude-devcontainer ~/.config/claude-devcontainer/

# 3. Install wrapper
cp bin/claude-docker.sh ~/.local/bin/claude-docker
chmod +x ~/.local/bin/claude-docker

# 4. Test it
make test

# 5. Add to your Emacs init (see Emacs section below)

# 6. Add token to your project
cd ~/your-project
cp /path/to/repo/.env.example .env   # paste your token
echo '.env' >> .gitignore
#+end_src

* Per-project .env

See [[file:.env.example][=./.env.example=]]
#+begin_src conf
# GitHub: https://github.com/settings/tokens (fine-grained, per-repo)
GITHUB_TOKEN=

# GitLab: https://gitlab.com/-/user_settings/personal_access_tokens
# GITLAB_TOKEN=

# Git identity
# GIT_AUTHOR_NAME=Your Name
# GIT_AUTHOR_EMAIL=you@example.com
#+end_src

* OAuth

The OAuth token is read from Keychain via:
#+begin_src sh
security find-generic-password -s 'Claude Code-credentials' -w 2>/dev/null
#+end_src

Fallback: =~/.config/claude-devcontainer/env=. Since credentials are stored in
clear in that file, the Keychain approach is strongly preferred.

*Less secure variant:* If you cannot use Keychain:
#+begin_src bash
claude setup-token
echo "CLAUDE_CODE_OAUTH_TOKEN=sk-ant-...." > ~/.config/claude-devcontainer/env
chmod 600 ~/.config/claude-devcontainer/env
#+end_src

* Git remote rewriting

When a git token is present in the project =.env=, SSH remote URLs are
automatically rewritten to HTTPS so Claude Code can push from inside the
container (which has no SSH client).

When =.env= is absent or contains no git token, rewriting is skipped
automatically. You can also control this explicitly:

#+begin_src conf
CLAUDE_NO_GIT_REWRITE=1   # always skip rewriting (even if a token is set)
CLAUDE_NO_GIT_REWRITE=0   # always rewrite (even without a token in .env)
#+end_src

Set this in your shell environment or in the project =.env=.

* Named instances

Run multiple Claude agents on the same codebase without conflicts.
Each named instance gets its own isolated state directory under
=~/.config/claude-devcontainer/instances/<name>/=.

#+begin_src sh
# Launch two agents in parallel
claude-docker -i stan
claude-docker -i kyle
#+end_src

Instance state is stored in:
#+begin_example
~/.config/claude-devcontainer/instances/
  stan/
    .seeded          # Sentinel (instance has been initialised)
    claude/          # Claude state (~/.claude)
    claude.json      # Claude config (~/.claude.json)
  kyle/
    ...
#+end_example

On first launch, credentials and config are copied from the host's
=~/.claude/= and =~/.claude.json= so the instance starts authenticated.
After that, each instance maintains its own state independently —
conversation history, project memory, and settings persist across runs.
Re-seeding on every launch would discard all of that accumulated state.

Stale instances (unused for 7 days) are pruned automatically. To
manually prune a specific instance:
#+begin_src sh
claude-docker -i kyle --prune
#+end_src

To re-seed an instance (e.g. after re-authenticating on the host):
#+begin_src sh
rm ~/.config/claude-devcontainer/instances/stan/.seeded
#+end_src

To list all instances with their associated project and status:
#+begin_src sh
claude-docker --list
#+end_src

#+begin_example
INSTANCE             STATUS     LAST USED            PROJECT
--------             ------     ---------            -------
stan                 running    2026-03-23 14:05     /home/user/project-a
kyle                 stopped    2026-03-21 09:30     /home/user/project-b
#+end_example

Without =--instance=, the default behaviour is unchanged: =~/.claude= is
shared and the concurrent-instance warning remains active.

* Network

No firewall — full network access. Protection is *filesystem isolation*: the
container only sees the current project directory.

* Emacs

=emacs/claude-code-ide-ext.el= extends
[[https://github.com/manzaltu/claude-code-ide.el][claude-code-ide]] with Docker wrapper support, named instances,
and an optional native (non-dockerized) mode.

** Installation

With =use-package= + =straight.el= (recommended):

#+begin_src emacs-lisp
(use-package claude-code-ide-ext
  :straight (:type git :host github :repo "jgru/claude-code-container"
             :files ("emacs/claude-code-ide-ext.el")))
#+end_src

Or load from a local checkout:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/claude-code-container/emacs")
(require 'claude-code-ide-ext)
#+end_src

** Commands

| Key / command                              | Description                                   |
|--------------------------------------------+-----------------------------------------------|
| =C-c C-'=                                 | Open Claude Code menu                         |
| =M-x claude-code-ide-ext-start=           | Start Claude with a named instance (completing)|
| =M-x claude-code-ide-ext-set-instance=    | Set instance name for the current buffer      |
| =M-x claude-code-ide-ext-list-instances=  | List all instances with status                |
| =M-x claude-code-ide-ext-prune-instance=  | Prune a stopped instance                      |
| =M-x claude-code-ide-ext-toggle-docker=   | Toggle between Docker and native CLI          |
| =M-x claude-code-ide-ext-init-env=        | Scaffold =.env= and add it to =.gitignore=   |

** Per-project instance name

Set a default instance per project via =.dir-locals.el=:

#+begin_src emacs-lisp
((nil . ((claude-code-ide-ext-instance . "stan"))))
#+end_src

** Customization

| Variable                              | Default                        | Description                      |
|---------------------------------------+--------------------------------+----------------------------------|
| =claude-code-ide-ext-docker-cli=      | =~/.local/bin/claude=          | Path to the Docker wrapper       |
| =claude-code-ide-ext-bare-cli=        | =claude=                       | Path to the native CLI           |
| =claude-code-ide-ext-use-docker=      | =t=                            | Use Docker wrapper when non-nil  |
| =claude-code-ide-ext-instances-dir=   | =~/.config/claude-devcontainer/instances/= | Instance state directory |
